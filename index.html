What is Leap?

Leap is a small programming language built to make data transformation code cleaner and safer. In most languages, chaining filters, maps, and transforms produces deeply nested calls or piles of temporary variables. Leap fixes that with one operator: |>.

Leap also makes errors impossible to ignore. Instead of crashing on bad input, functions return Ok(value) or Err(message). The ? operator threads errors through a pipeline automatically, so you never have to write repetitive error-checking code.

DataPipeline.leap
let scores = [88.5, 72.0, 95.3, 61.1, 44.8, 90.0]

let result = scores
    |> filter(fn(s) { s >= 60.0 })
    |> map(fn(s) { s * 1.05 })

print(str(result))
python3 leapinterpreter.py DataPipeline.leap
Errorpipeline.leap
fn safeDivide(a, b) {
    if b == 0 { Err("divide by zero") } else { Ok(a / b) }
}

fn main() {
    let result =
        10
        |> fn(x) { safeDivide(x, 2) }?
        |> fn(x) { Ok(x + 5) }?
    Ok(result)
}

print(str(main()))   -- Ok(10)
python3 leapinterpreter.py Errorpipeline.leap

How the interpreter works

01
Lexer
Reads source text character by character and produces a flat list of tokens.
02
Parser
Reads tokens and builds a syntax tree using recursive descent.
03
Evaluator
Walks the tree node by node and computes the final result.

What Leap supports

  • |> pipe operator -- chain transformations left to right with no nesting
  • Ok / Err -- safe error handling built into the language
  • ? operator -- automatically propagates errors through a pipeline
  • fn -- named and anonymous functions, closures included
  • if / else as expressions -- assign the result of a conditional directly
  • let bindings -- clean, immutable variable declarations
  • map / filter / range -- built-in list operations for pipeline work
